# Project Instructions and Guidelines

This document contains important rules and guidelines for developing in this project.

## Core Development Rules

1.  **No Middleware**: Do not use Next.js middleware or similar middleware patterns unless absolutely necessary and explicitly approved. Prefer other architectural patterns for handling requests and authentication.
2.  **No Deprecated Dependencies**: Do not use deprecated packages or dependencies. Always check the status of a package before adding it.
3.  **No Unsecure Dependencies**: Do not use packages with known security vulnerabilities. Regularly audit dependencies for security issues.
4.  **Resolve All Problems and Warnings**: Always resolve all linter warnings, compiler errors, and other reported problems. Do not ignore them.
