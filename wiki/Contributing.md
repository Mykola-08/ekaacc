# Contributing

Thanks for contributing! This document outlines how to contribute to the project.

- Fork the repo and create a feature branch from `main`.
- Run tests locally: `npm test`.
- Follow code style and run linters if available.
- Open a Pull Request with a clear description and link to any relevant docs.
- For database or schema changes, include migration files under `supabase/migrations/` and update `DATABASE_MODIFICATIONS.md`.

Useful references:
- `README.md` - Project overview and local dev
- `FULL_IMPLEMENTATION_GUIDE.md` - Roadmap and larger changes
- `TESTING.md` - Running and writing tests

If you're unsure where to start, check the `good first issue` label in GitHub issues.
